easyunfold (ABINIT Extension)

The original easyunfold supports VASP and CASTEP. This repository extends its capabilities to fully support ABINIT calculations.

🚀 Quick Start & Workflow

1. Install the Modified Files

First, copy the modified files into your local easyunfold installation directory:

SITE=$(python3 -c 'import easyunfold; import os; print(os.path.dirname(easyunfold.__file__))')
cp abinit_wfk.py wavefun.py unfold.py utils.py $SITE/
find $SITE -name '*.pyc' -delete


2. Initialise easyunfold

Generate the easyunfold.json configuration and the supercell k-points file (KPOINTS). You will need POSCAR files for both your primitive cell and supercell, along with a high-symmetry k-path text file.

easyunfold generate prim.POSCAR sc.POSCAR kpath.txt --code abinit


⚠️ Important: You must include the --code abinit flag. This tells easyunfold which DFT code to configure and saves this preference in easyunfold.json, which must remain in your working directory.

3. Convert KPOINTS to ABINIT Format

The generated KPOINTS file is in VASP format. You can manually format these into your ABINIT input, or use this quick Python script to extract and format them:

kpts = []
with open('KPOINTS') as f:
    lines = f.readlines()
    nk = int(lines[1])
    for line in lines[3:3+nk]:
        k = line.split()[:3]
        kpts.append(f'  {k[0]}  {k[1]}  {k[2]}')

print(f'nkpt  {len(kpts)}')
print('kpt')
print('\n'.join(kpts))
print(f'wtk  {len(kpts)}*1.0')


Copy the output of this script into your non-SCF ABINIT input file (e.g., trf2_2.abi).

4. Run ABINIT Calculations

You will need a two-step ABINIT calculation process:

Run 1: SCF Calculation
Run a standard self-consistent field (SCF) calculation to generate the density file (e.g., trf2_1o_DEN).

Run 2: Non-SCF Band Structure
Run a non-self-consistent calculation to compute eigen energies. Ensure your input file contains the following parameters:

iscf       -2     # Non-self-consistent calculation for eigenvalue computation
getden     -1     # Get charge density from the previous run (e.g., inputname_1i_DEN)
kptopt      0     # Read directly nkpt and kpt lists
nkpt        x     # (Number of k-points from Step 3)
kpt   list[x]     # (List of k-points from Step 3)
tolwfr      1.0d-12
nstep       50
prtden      1
istwfk     *1     # REQUIRED: Do NOT take advantage of time-reversal symmetry
iomode      3     # REQUIRED: Use NetCDF library to produce ETSF specification files
prteig      1
prtwf       1     # REQUIRED: Print wavefunctions


This generates the wavefunction file: trf2_2o_WFK.nc.

(Optional) Verify coefficients were written correctly:

import netCDF4 as nc, numpy as np
ds = nc.Dataset('trf2_2o_WFK.nc', 'r')
raw = ds.variables['coefficients_of_wavefunctions']
filled = np.ma.filled(raw[0,5,4,0,:3,0], 0.0)
masked = np.ma.is_masked(raw[0,5,4,0,:3,0])
print('masked:', masked, '  values:', filled)
ds.close()


Expected output: masked: False followed by non-zero, non-uniform values.

5. Calculate Spectral Weights

Read the WFK file, compute the spectral weight $P(K,E)$ for each supercell k-point by projecting wavefunction coefficients onto the primitive cell G-sphere, and save the results.

rm -f easyunfold.json.bak  # Optional backup
easyunfold unfold calculate trf2_2o_WFK.nc


6. Plot the Unfolded Band Structure

Plot your unfolded bands using the calculated data.

easyunfold unfold plot \
    --eref <fermi_energy_in_eV> \
    --emin -5 --emax 9 \
    --intensity 10 \
    -o unfolded_bands.png


💡 Finding the Fermi Energy: The --eref value should be the Fermi energy from your SCF run (e.g., printed in trf2_1.abo or read from the DEN file). To extract it automatically from the WFK:

import netCDF4 as nc, numpy as np
ds = nc.Dataset('trf2_2o_WFK.nc','r')
ef = float(np.asarray(ds.variables['fermi_energy']).flat[0]) * 27.211386
print(f'Fermi energy: {ef:.6f} eV')
ds.close()


🛠️ Troubleshooting

Symptom

Cause

Fix

coefficients_of_wavefunctions all masked

ABINIT compiled with MPI-IO, or prtwf not set.

Recompile ABINIT without MPI-IO. Add prtwf 1 and iomode 3 to the Run 2 input.

G-vector count mismatch error

istwfk not set to 1 for all k-points.

Add istwfk *1 to the input. This forces the full G-sphere (no time-reversal compression).

All spectral weights uniform (~1/nplw)

G-vector ordering mismatch between reconstruction and internal order.

The KE-sorted reconstruction should match. Verify ecut matches between runs.

VBM/CBM nonsensical (e.g. CBM < VBM)

All occupancies = 2.0 in non-SCF run.

Handled by get_vbm_cbm fallback to fermie. Use --eref explicitly in the plot command.

Plot shows all bands equally bright

Old easyunfold.json cached from a broken run.

Delete easyunfold.json and rerun easyunfold unfold calculate.

K-points mismatch error

kptopt not set to 0, or kpt list has wrong values.

Use kptopt 0 with the exact k-points from the KPOINTS file generated by easyunfold.

🔍 Under the Hood: Code Modifications

This extension introduces several precise modifications to integrate ABINIT into the easyunfold ecosystem smoothly.

1. New File: abinit_wfk.py

Provides the AbinitWavefunction class to read ABINIT NetCDF WFK files. The public interface deliberately mirrors the Wavecar class.

Constants: Conversion factors added (HARTREE_TO_EV, BOHR_TO_ANG, TPI).

Design Decisions:

Self-Contained Initialization: Reads everything on __init__ and closes the file immediately to match Wavecar behavior.

G-vector Reconstruction: Uses the exact KE filter as VASP's get_gvectors (in Hartree/Bohr atomic units). It tries the file first but reconstructs if ABINIT sentinel placeholders (INT32_MIN) are found. It sorts by ascending KE for deterministic ordering.

Safe Coefficient Reading: Uses np.ma.filled(..., 0.0) instead of np.array to correctly replace netCDF4-python MaskedArray paddings with zero, preserving real data.

fermie Property: Non-SCF runs have occupancies forced to 2.0. Reading fermi_energy directly from the WFK (converted Ha $\to$ eV) provides a reliable energy reference.

2. wavefun.py Modifications

Imported the new reader: from .abinit_wfk import AbinitWavefunction as WFKReader.

Added the AbinitWaveFunction wrapper class. It delegates every property (kpoints, nspins, mesh_size, bands, occupancies, fermie) and method (get_gvectors, get_band_coeffs) to the underlying reader so that VASP, CASTEP, and ABINIT are fully interchangeable inside unfold.py.

3. unfold.py Modifications

Initialization (__init__): Added the abinit branch to route parsing through AbinitWaveFunction.from_file(fname, lsorbit=self._lsoc).

Fixed get_vbm_cbm for Non-SCF: Updated the VBM/CBM detection logic to fall back to the WFK's Fermi energy if all bands report as occupied (a quirk of ABINIT iscf = -2 runs reading from density).

4. utils.py Modifications

write_kpoints: Added an abinit dispatch that writes k-points in VASP KPOINTS format (which users adapt for the ABINIT input).

read_kpoints: Added an abinit dispatch to read VASP-format KPOINTS files.
